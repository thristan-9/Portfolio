About me

My first contact with code was 5 years ago as an intern at a startup called Clinicorp, a Saas company with more than 14,000 users that helps clinics self-manage. When I joined the company, there were around 15 employees in the entire company and that was great for me, since there was almost no development process and everything was very rustic, I had to learn a bit of everything before I could start delivering value to the company, technologies like HTMl, Css, Javascript React, a bit of DevOps and Git and then Node js. But it did not take too long because I had the opportunity to learn and work with very good developers.

Experiences

During this 5 years, I was responsible for many full stack projects such as recurring billing, import of financial data, a bank account project and much more. And after a while, I began to realize that just seeing URLs in the postman, SQL queries and terminals didn't captivate me as much as creating beautiful interfaces with code and a good user experience at the same time.

One of the best experiences I had at Clinicorp was planning and creating a complex interface that allowed clinics to set up a bank account within the system. This allowed them to integrate different payment methods directly into the Clinicorp system. The client had to send some documents and data from clinic and partners through the system, a team validated the data and documents through an internal platform, and if everything was in order we sent the data to Dock, a white label banking company(Baas), once the account was created the client could manage it through the Clinicorp app. I was responsible for the frontend of this project and two other developers were responsible for the backend. In the beginning it was very challenging as there were some things I had never done before, but the project started to structure itself and in the end it was an incredible feeling to see everything working as it should.
